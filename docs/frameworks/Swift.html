Swift

Swift es el lenguaje de programación moderno de Apple, diseñado para ser seguro, rápido y expresivo. Es el lenguaje principal para desarrollar aplicaciones en iOS, macOS, watchOS y tvOS.

1. Constantes y Variables​

En Swift, las constantes y variables son dos estructuras fundamentales para almacenar y manipular datos en tu código. La elección de una u otra depende de si el valor almacenado cambiará o permanecerá constante a lo largo de la ejecución del programa.

1.1 Constantes let​

Una constante en Swift es una entidad cuyo valor no puede cambiar una vez que se le ha asignado. Se define con la palabra clave let.

let nombreDeLaConstante = valor
let nombre = "Juan"
let pi = 3.14159

Uso: Las constantes son útiles para almacenar datos que no cambiarán durante la ejecución del programa, como valores matemáticos (pi), datos de configuración, o datos de un usuario que no se modificarán.

Ventajas:

  • Seguridad — Al ser inmutables, garantizan que los valores no se modifiquen accidentalmente.
  • Optimización — Swift puede optimizar el uso de memoria para constantes, ya que el valor no se actualiza.

1.2 Variables var​

Una variable es una entidad cuyo valor puede cambiar a lo largo de la ejecución del programa. Se define con la palabra clave var.

var nombreDeLaVariable = valor
var edad = 25
edad = 26 // Se permite cambiar el valor

Uso: Las variables son útiles para valores que se espera que cambien, como el puntaje en un juego, el saldo de una cuenta, o datos temporales que puedan requerir modificaciones.

Ventajas:

  • Flexibilidad — Permite almacenar datos dinámicos o temporales.
  • Interactividad — Ideal para valores que responden a acciones del usuario o datos en tiempo real.

1.3 Diferencias clave entre Constantes y Variables​

CaracterísticaConstante (let)Variable (var)
MutabilidadInmutable (no cambia)Mutable (puede cambiar)
Uso de memoriaOptimizadoPuede ocupar más memoria
Sintaxislet nombre = valorvar nombre = valor

1.4 Mutación de datos​

Se refiere a cambiar o modificar el valor de una variable después de que ha sido creada. La mutación solo es posible en variables (var), ya que las constantes (let) no permiten cambios tras su inicialización.

var nombre = "Ana"       // variable mutable
nombre = "Luis"          // mutación: se cambia el valor

let apellido = "Gómez"   // constante inmutable
apellido = "Martínez"    // error: las constantes no pueden mutarse

1.5 Declaración múltiple​

Para declarar varias variables o constantes en una sola línea, puedes separar los valores con comas:

var x = 5, y = 10, z = 15

O usar el punto y coma para separar múltiples declaraciones en una misma línea:

let a = 10; let b = 20; let c = a + b
tip

Aunque es posible usar el punto y coma, en Swift es más común y recomendado escribir una declaración por línea para mejorar la legibilidad del código.

1.6 Nombrar Constantes y Variables​

En Swift, los nombres pueden incluir casi cualquier carácter, incluyendo Unicode:

let π = 3.14159
let 你好 = "你好世界"
let 🐶🐮 = "dogcow"

Reglas:

  • No deben incluir espacios en blanco, símbolos matemáticos ni flechas.
  • No pueden comenzar con un número.
  • No puedes volver a declarar una constante o variable con el mismo nombre.
  • No puedes cambiarla para almacenar valores de un tipo diferente.
tip

Si necesitas usar una palabra clave reservada de Swift como nombre, puedes encerrarla en comillas invertidas. Sin embargo, se recomienda evitarlo.

let `class` = "Este es un ejemplo"

2. Tipos de datos​

Tanto las constantes como las variables en Swift pueden almacenar cualquier tipo de datos, ya sea primitivo (Int, Float, Double, String, Bool) o complejo (Array, Dictionary, Set, Struct, Class).

2.1 Enteros Int​

Los enteros representan números sin decimales, positivos o negativos. Swift ajusta automáticamente el tamaño según la arquitectura del dispositivo (32 o 64 bits).

let edad: Int = 30
var diasEnUnAno = 365

Límites: Swift proporciona Int.min y Int.max para obtener los límites mínimo y máximo.

Subtipos de Enteros​

TipoDescripción
Int8 / UInt8Enteros de 8 bits con/sin signo
Int16 / UInt16Enteros de 16 bits
Int32 / UInt32Enteros de 32 bits
Int64 / UInt64Enteros de 64 bits
let numeroPequeño: Int8 = -120
let numeroGrande: UInt64 = 18446744073709551615

2.2 Decimales Double y Float​

TipoBitsPrecisión
Double64 bitsHasta 15-16 decimales
Float32 bitsHasta 6-7 decimales
let pi: Double = 3.14159265358979
let temperatura: Float = 21.5

Swift infiere automáticamente Double para valores decimales si no se especifica el tipo.

Conversión de enteros a decimales​

var numeroEntero: Int = 42
var numeroDouble: Double = Double(numeroEntero)
// numeroDouble == 42.0

2.3 Cadenas de texto String​

String es el tipo de datos para manejar texto. Permite cualquier secuencia de caracteres, palabras, frases y símbolos.

let saludo: String = "Hola, mundo!"
var nombre: String = "Carlos"

Interpolación de cadenas:

let edad = 25
let mensaje = "Tengo \(edad) años"

Cadenas multilínea:

let poema = """
Este es un poema
que tiene varias líneas.
"""

2.4 Booleanos Bool​

Bool representa valores de verdad: true o false. Se usa para manejar condiciones y control de flujo.

let esMayorDeEdad: Bool = true
var estaLloviendo = false

Los valores booleanos se usan en declaraciones if, while y for para controlar la lógica del programa.

2.5 Asignación múltiple​

Puedes declarar variables o constantes que comparten el mismo tipo asignando el tipo al final:

var x, y, z: Int
x = 5
y = 10
z = 15

3. Comments​

Los comentarios se usan para agregar anotaciones al código que no afectan su ejecución, pero sirven para explicar o documentar lo que hace el código, lo que facilita su comprensión.

Comentarios de una sola línea:

Se usan con dos barras //. Todo lo que esté a la derecha de // en esa línea será ignorado por el compilador.

// Este es un comentario de una sola línea

let numero = 42  // También puedes escribir un comentario al final de una línea de código

Comentarios multilínea:

Se utilizan cuando el comentario ocupa varias líneas. Se abre con /* y se cierra con */.

/*
Este es un comentario que puede
ocupar varias líneas.
Es útil para explicaciones más largas.
*/

También puedes usar /* para comentar parte de una línea de código:

let a = 10 /* esto es un comentario dentro de una línea */

Propiedades y Usos de los Comentarios:

  • Documentación: Los comentarios son útiles para documentar funciones o algoritmos complejos, explicando lo que hace cada parte del código.
  • Desactivar código temporalmente: Puedes comentar una sección del código para desactivarlo sin eliminarlo. Útil al probar diferentes implementaciones o desactivar funcionalidades durante la depuración.
  • Mejorar la legibilidad: Ayudan a que otras personas (o incluso tú mismo en el futuro) comprendan el propósito y la lógica detrás de ciertas secciones del código.

4. Print​

El comando print se usa para enviar información a la consola, lo cual es muy útil para depurar tu código, verificar valores de variables, el flujo del programa o cualquier mensaje de interés.

print("Texto a mostrar")

Esto imprimirá el texto "Texto a mostrar" en la consola. Además, se pueden imprimir variables y expresiones dentro de print:

let nombre = "Juan"
print("Hola, \(nombre)")  // Hola, Juan

El uso de \(nombre) es interpolación de cadenas — puedes insertar el valor de una variable o expresión dentro de una cadena de texto.

Propiedades y Uso:

  • Debugging: Usamos print para ver el valor de variables en diferentes puntos del código.
  • Seguimiento del flujo: Puedes agregar print en varias secciones del código para ver cómo se ejecuta.
  • Imprimir datos complejos: Puedes imprimir cualquier tipo de datos (arrays, diccionarios, objetos) y Swift mostrará su descripción automáticamente.
let numeros = [1, 2, 3, 4]
print(numeros)  // [1, 2, 3, 4]

Opciones adicionales:

  • separator: Define un string que separa los elementos cuando hay más de uno. Por defecto es un espacio.
print(1, 2, 3, separator: "-")  // 1-2-3
  • terminator: Especifica el carácter al final de la impresión. Por defecto es una nueva línea (\n).
print("Hola", terminator: "")  // No imprime un salto de línea al final
print("Mundo")                  // Continúa en la misma línea: "HolaMundo"

5. Typealias​

En Swift, los typealiases (alias de tipo) son una forma de asignar un nombre más legible o más conveniente a un tipo existente. Un typealias no crea un tipo nuevo, sino que es simplemente un sinónimo del tipo original. Su uso principal es mejorar la legibilidad y la claridad del código, o proporcionar una forma más concisa para referirse a tipos complejos.

Sintaxis de un Typealias

typealias Nombre = String
let userName: Nombre = "Nombre reemplaza al Tipo String"

Propiedades de los Typealiases

  • No crea un tipo nuevo: Un typealias no crea un tipo nuevo, solo es un alias. El tipo original sigue existiendo, pero puedes referirte a él con un nombre diferente.
  • Alias de tipos complejos: Son especialmente útiles para simplificar tipos complejos o largos, como los tipos de funciones, genéricos, o tipos de colecciones anidadas.
  • Uso en funciones, estructuras, y clases: Los typealiases se pueden utilizar en cualquier parte del código: dentro de funciones, estructuras, clases, o incluso de manera global.
  • El tipo original se mantiene: Los alias no afectan el comportamiento del tipo original, ni lo hacen más flexible.

¿Cuándo usarlos?

  • Cuando el nombre del tipo original es largo o complicado.
  • Cuando quieras mejorar la legibilidad del código.
  • Para simplificar la declaración de tipos complejos que se usan varias veces en el código.

Limitaciones

  • No puedes usar typealiases para modificar el comportamiento de un tipo. Un alias es solo un nombre alternativo, el tipo original sigue siendo el mismo.
  • No puedes crear typealiases para tipos que no sean tipos de datos (como funciones, protocolos, etc.).

5.1 Typealias en una Tuple​

Tipo de tuples como alias: Supón que tienes una tuple que contiene un Int y un String, y la usas en varias partes del código. Un typealias puede ayudar a que tu código sea más expresivo.

typealias Person = (id: Int, name: String)

let person1: Person = (id: 1, name: "John")
let person2: Person = (id: 2, name: "Alice")

5.2 Typealias en colecciones​

Si trabajas con un conjunto de tipos complejos, los typealiases pueden ser útiles para hacer que las colecciones sean más legibles.

typealias ProductList = [Int: String]

Los typealiases también se pueden usar con tipos genéricos para simplificar la definición de tipos complejos. Por ejemplo, si estás trabajando con un diccionario de clave-valor donde la clave es un String y el valor es una lista de enteros:

typealias DictionaryOfLists = [String: [Int]]

let myData: DictionaryOfLists = [
    "key1": [1, 2, 3],
    "key2": [4, 5, 6]
]

📎 typealias DictionaryOfLists = [String: [Int]] define un alias de tipo. [String: [Int]] es un diccionario donde la clave es de tipo String y el valor es un array de enteros [Int]. let myData: DictionaryOfLists = [...] declara una constante de ese tipo con valores concretos.

5.3 Typealias de tipo de cierre (Closures)​

Imagina que tienes una función que acepta un tipo de cierre muy largo y complicado:

let completion: (Bool, String) -> Void = { success, message in
    print("Success: \(success), Message: \(message)")
}

Puedes crear un typealias para hacer que el código sea más legible:

typealias CompletionHandler = (Bool, String) -> Void

let completion: CompletionHandler = { success, message in
    print("Success: \(success), Message: \(message)")
}

📎 typealias CompletionHandler = (Bool, String) -> Void crea un alias para un tipo de función que toma un Bool y un String como parámetros y devuelve Void. completion es una closure de ese tipo donde success indica si la operación fue exitosa y message aporta detalles del resultado.

5.4 Typealias para tipos con Protocolos​

Si estás trabajando con protocolos o combinaciones de tipos que pueden ser largos, puedes usar typealiases para hacer que el código sea más limpio:

protocol Drawable {
    func draw()
}

typealias DrawableObject = Drawable & CustomStringConvertible

struct Circle: DrawableObject {
    var description: String {
        return "Circle"
    }

    func draw() {
        print("Drawing a circle")
    }
}

6. Tuples​

Las tuples en Swift son una forma de agrupar varios valores en un solo valor compuesto. A diferencia de los arrays o diccionarios, las tuples pueden contener valores de diferentes tipos.

let persona = ("Juan", 30, "España")

print(persona.0) // Juan
print(persona.1) // 30
print(persona.2) // España

6.1 Tuples con nombres para los elementos​

Una de las grandes ventajas de las tuplas en Swift es que puedes darle nombres a cada uno de los valores que contiene. Esto hace que el código sea más claro y legible:

let persona = (nombre: "Juan", edad: 30, pais: "España")
print(persona.nombre) // Juan
print(persona.edad)   // 30
print(persona.pais)   // España

6.2 Usos y utilidades de las Tuples​

  • Devolver múltiples valores de una función: una de las aplicaciones más comunes de las tuples es devolver varios valores de una función sin tener que crear una estructura personalizada.
func obtenerDimensiones() -> (ancho: Int, alto: Int) {
    return (1024, 768)
}

let dimensiones = obtenerDimensiones()
print(dimensiones.ancho) // 1024
print(dimensiones.alto)  // 768
  • Agrupar datos temporalmente: las tuples son ideales cuando necesitas agrupar datos de manera temporal.
  • Trabajar con patrones de correspondencia (pattern matching): Swift permite descomponer las tuples mediante pattern matching, útil en for, switch y otras estructuras de control.
let coordenada = (x: 10, y: 20)

switch coordenada {
case (0, 0):
    print("Origen")
case (let x, 0):
    print("En el eje X en \(x)")
case (0, let y):
    print("En el eje Y en \(y)")
case (let x, let y):
    print("En algún lugar en (\(x), \(y))")
}
  • Comparar valores fácilmente: Puedes comparar tuples que tengan el mismo tipo de elementos.
let punto1 = (x: 10, y: 20)
let punto2 = (x: 10, y: 20)

if punto1 == punto2 {
    print("Los puntos son iguales")
}

6.3 Ventajas y limitaciones de las Tuples​

Ventajas:

  • Son fáciles de usar y permiten agrupar valores sin crear nuevas clases o estructuras.
  • Son útiles para funciones que necesitan devolver múltiples valores rápidamente.
  • Pueden mejorar la legibilidad cuando se usan para datos de corto plazo.

Limitaciones:

  • No están pensadas para datos complejos o que necesiten persistir durante un tiempo prolongado.
  • Las tuples no tienen métodos o funcionalidades avanzadas como las estructuras o clases.
  • No se puede modificar el tipo o cantidad de elementos una vez que una tuple ha sido creada.

¿Cuándo usar las Tuples? Las tuples son perfectas para casos simples y de corto plazo. Si necesitas representar una estructura de datos más compleja o que tenga lógica adicional, es mejor usar struct o class.

7. Optionals​

En Swift, los opcionales son un tipo de datos que permiten que una variable o constante tenga un valor o no. Pueden almacenar un valor válido o representar la ausencia de valor. Los opcionales son muy importantes en Swift porque manejan de manera segura la posibilidad de que algo no exista, evitando errores comunes de referencia nula.

¿Qué es un opcional?

Un opcional en Swift es un tipo de variable que puede contener un valor de un tipo específico o puede no tener ningún valor en absoluto. El tipo de un opcional se declara agregando un signo de interrogación (?) al tipo de datos.

var nombre: String?  // `nombre` puede ser un String o nil
nombre = "Juan"      // Asignamos un valor
nombre = nil         // O podemos asignar nil

Propiedades y características de los opcionales

  • Valor o nil: Un opcional puede tener un valor o estar vacío (representado por nil).
  • Tipo de datos opcionales: Un tipo opcional se define con ?. Ejemplo: String? indica un tipo que puede ser un String o nil.

Desempaquetado (unwrapping)

Para acceder al valor de un opcional, primero debes desempaquetarlo. Hay varias formas:

  • Desempaquetado forzado (!): Si estás seguro de que el opcional tiene un valor.
var nombre: String? = "Juan"
print(nombre!)  // Imprimirá "Juan"

⚠️ Si el opcional es nil y se intenta desempaquetar de esta manera, el programa generará un error en tiempo de ejecución.

  • Desempaquetado seguro (if let y guard let): Solo se ejecuta si el opcional tiene un valor.
var nombre: String? = "Juan"

if let nombreDesempaquetado = nombre {
    print("El nombre es \(nombreDesempaquetado)")
} else {
    print("No hay valor para el nombre")
}
func saludar() {
    guard let nombreDesempaquetado = nombre else {
        print("No hay valor para el nombre")
        return
    }
    print("Hola, \(nombreDesempaquetado)")  // "Hola, Juan"
}
saludar()
  • Desempaquetado implícito (! en la declaración): Cuando estás seguro de que un opcional tendrá valor tras ser inicializado.
var nombre: String! = "Juan"
print(nombre)  // No es necesario desempaquetar
  • Nil-Coalescing Operator (??): Proporciona un valor predeterminado si el opcional es nil.
var nombre: String? = nil
let nombreFinal = nombre ?? "Desconocido"
print(nombreFinal)  // "Desconocido"

7.1 Tipos de opcionales en Swift​

  • Opcionales simples (?): Como String?, Int?, etc.
var objeto: MyClass?
  • Tipos con valores predeterminados: Al usar el operador nil-coalescing puedes proporcionar un valor predeterminado cuando un opcional es nil.

Uso común de los opcionales

  • Manejo de errores: Cuando trabajas con datos que pueden ser opcionales (consultas a base de datos, datos de una API).
  • Interacción con interfaces gráficas: Campos de texto que podrían estar vacíos.
  • Almacenamiento en colecciones: Arrays o diccionarios con valores opcionales.
var edades: [String: Int?] = ["Juan": 30, "Ana": nil]

Resumen

  • Los opcionales representan la posibilidad de que una variable no tenga valor (nil).
  • Se declaran con ? (por ejemplo, String?).
  • Se desempaquetan de forma forzada (!), segura (if let, guard let), o implícita (String!).
  • El operador ?? permite un valor predeterminado si el opcional es nil.

7.2 Custom Optional​

El tipo CustomOptional te permite declarar una variable como opcional personalizado usando enumeradores:

enum CustomOptional<Wrapped> {
    case none
    case some(Wrapped)
}

let myOptionalInt = CustomOptional<Int>.none
let myInt = CustomOptional<Int>.some(10)

print(myOptionalInt)   // none  - estado de nil
print(myInt)           // some(10) - un posible valor de 10
